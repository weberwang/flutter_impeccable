Improve Flutter UX copy, labels, validation, and state messages.

## Targets

- App bars and page titles.
- Button labels and menu items.
- Form labels, helper text, and errors.
- Empty, loading, offline, permission, and success states.
- SnackBars, dialogs, bottom sheets, and confirmations.
- Semantics labels and tooltips.

## Rules

- Use the project's localization layer when present.
- Keep labels action-oriented and specific.
- Place errors near the related field.
- Explain recovery, not just failure.
- Avoid concatenated translation fragments.
- Ensure icon-only controls have meaningful labels.
